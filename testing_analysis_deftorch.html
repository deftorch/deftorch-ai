Temuan & masalah utama

KRITIS

model-selector.test.ts menguji model yang tidak ada

MODEL_BUTTON_REGEX = /Kimi|Codestral|Mistral|DeepSeek|GPT|Grok/i — tapi models.ts hanya berisi Gemini 2.5/3.x. Test ini akan gagal di app nyata karena mencari "Mistral Small" dan grup "Moonshot" yang tidak terdaftar.

KRITIS

AI API key tidak ada di CI workflow

playwright.yml hanya set BETTER_AUTH_SECRET, POSTGRES_URL, BLOB_READ_WRITE_TOKEN, REDIS_URL. Tidak ada GOOGLE_GENERATIVE_AI_API_KEY. Test di api.test.ts yang mengirim pesan dan menunggu respons AI akan gagal di CI.

PERINGATAN

lib/ai/models.test.ts bukan file test

File bernama *.test.ts tapi tidak ada satupun test() atau describe() di dalamnya — hanya ekspor mock model factory. Ini membingungkan dan berbeda dari models.mock.ts yang melakukan hal serupa dengan pendekatan berbeda (dua sistem mock yang redundan).

PERINGATAN

Fixtures dan Page Object didefinisikan tapi tidak pernah dipakai

tests/fixtures.ts mengekspor custom test dengan chatPage fixture, dan tests/pages/chat.ts mengimplementasikan ChatPage POM — tapi keempat file E2E mengimpor langsung dari @playwright/test, bukan dari fixtures. helpers.ts (generateRandomTestUser, generateTestMessage) juga tidak pernah diimpor di mana pun.

PERINGATAN

Test flaky dengan catch kosong

Di chat.test.ts: await stopButton.click({timeout:5000}).catch(() => {}) — test selalu lulus tanpa memvalidasi apa pun. Komentar dalam kode: "Generation may have finished before we could click" mengakui ini adalah non-assertion.

PERINGATAN

Konstanta duplikat

MODEL_BUTTON_REGEX didefinisikan identik di dua file: tests/e2e/model-selector.test.ts dan tests/pages/chat.ts. Seharusnya diekspor dari satu tempat.

INFO

Timeout 240 detik terlalu tinggi, 0 retries terlalu rendah

Kombinasi 240s timeout + 0 retries tidak ideal untuk E2E. Timeout besar mengindikasikan test lambat (kemungkinan karena hit API nyata). E2E test biasanya butuh retry 1–2x untuk mengatasi flakiness jaringan, terutama di CI.

INFO

Tidak ada unit test sama sekali

Tidak ada Vitest atau Jest. Logika kritis seperti lib/db/queries.ts, lib/ai/tools/, lib/ai/rag.ts, lib/editor/, lib/ai/gemini.ts, dan server actions tidak dicakup sama sekali.
